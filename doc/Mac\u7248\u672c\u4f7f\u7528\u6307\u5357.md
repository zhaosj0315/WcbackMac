# Mac 版本使用指南

## 1. 前置条件

- macOS 12+，微信 4.x（已登录）
- Python 3.10+
- Xcode Command Line Tools（提供 LLDB）

```bash
pip install -r requirements.txt
```

---

## 2. 完整流程

### 步骤 1：探测环境

```bash
python3 scripts/mac_probe_wechat.py
```

确认微信进程、沙盒目录和数据库路径。

### 步骤 2：解密数据库

```bash
python3 scripts/mac_auto_decrypt_export.py --quit-original
```

流程：退出微信 → 复制副本 → ad-hoc 重签 → 启动副本 → LLDB 扫描 key → 批量解密到 `app/DataBase/MacMsg/`。

已有 key 文件时可跳过 LLDB：

```bash
python3 scripts/mac_decrypt_from_keys.py \
  --keys /tmp/wechat_lldb_key_candidates.json \
  --output app/DataBase/MacMsg --verify
```

### 步骤 3：生成联系人映射

```bash
cd scripts
python3 mac_contact_mapper.py
```

输出 `data/mac_contact_mapping.json`，后续导出和 Web 查看器均依赖此文件。

### 步骤 4a：Web 查看器（推荐）

```bash
cd scripts
python3 mac_web_server.py
# 访问 http://127.0.0.1:5000
```

功能：
- 左侧会话列表，支持搜索
- 消息类型筛选（文字/图片/语音/视频/文件/表情）
- 正序/倒序切换
- 语音在线播放、视频在线播放、文件下载
- 点击顶部统计栏查看联系人列表和朋友圈

### 步骤 4b：命令行全量导出

```bash
cd scripts

# 全量导出（含媒体文件复制）
python3 mac_export_by_session.py --output ../data

# 只导出聊天记录，跳过媒体复制（快很多）
python3 mac_export_by_session.py --output ../data --no-global-media

# 单个会话测试
python3 mac_export_by_session.py --wxid wxid_xxx --output ../data --no-global-media
```

## 4. 技术说明

### 消息解码

Mac 微信 4.x 消息内容使用 **zstd 压缩**（魔数 `0x28B52FFD`），格式为：

```
wxid:\n<正文或XML>
```

### 数据库分片

消息按年份分片：`message_0.db`（最新）到 `message_9.db`（最旧）。每个分片有独立的 `Name2Id` 表，同一联系人在不同分片中的 `real_sender_id` 不同，导出时按分片独立映射。

### 媒体定位

- 图片/视频：先查 `MessageTemp/<会话md5>/Image(Video)/`，找不到按 `create_time` 降级匹配，再兜底查 `attach/<会话md5>/` 目录
- 语音：从 `media_0.db` 的 `VoiceInfo` 表提取 silk 数据，自动转 wav

## 5. 常见问题

**Q：为什么有些图片显示为占位？**  
本地 `MessageTemp` 缓存已被微信清理，或原图未下载。`attach/` 目录兜底也找不到时显示占位，属正常现象。

**Q：为什么有些视频无法播放？**  
视频文件只在微信下载后才存在本地缓存，未下载的视频无法播放。

**Q：FTS 数据库完整性检查失败？**  
`*_fts.db` 是全文检索索引，不影响消息导出。

**Q：my_wxid 如何确定？**  
工具自动从 `xwechat_files/wxid_*` 目录名检测，也可通过 `--my-wxid` 参数手动指定。
